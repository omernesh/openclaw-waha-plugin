**In src/directory.ts**, enhance `getGroupParticipants()` to JOIN with contacts for @lid name resolution:

Replace the current SQL query in `getGroupParticipants()` with one that LEFT JOINs the contacts table to resolve @lid participant JIDs to their @c.us display names:

```sql
SELECT gp.group_jid, gp.participant_jid,
  COALESCE(gp.display_name, c_direct.display_name, c_cus.display_name) as display_name,
  gp.is_admin, gp.allow_in_group, gp.allow_dm, gp.participant_role
FROM group_participants gp
LEFT JOIN contacts c_direct ON gp.participant_jid = c_direct.jid
LEFT JOIN contacts c_cus ON REPLACE(gp.participant_jid, '@lid', '@c.us') = c_cus.jid
  AND gp.participant_jid LIKE '%@lid'
WHERE gp.group_jid = ?
ORDER BY display_name ASC, gp.participant_jid ASC
```

This query:
1. First tries `gp.display_name` (already stored in group_participants)
2. Falls back to `c_direct.display_name` (direct JID match in contacts)
3. Falls back to `c_cus.display_name` (for @lid JIDs, tries the @c.us equivalent in contacts)

The `c_cus` JOIN only activates for @lid JIDs (the `LIKE '%@lid'` condition prevents unnecessary joins for @c.us JIDs).
