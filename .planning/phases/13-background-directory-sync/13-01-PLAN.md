```sql
CREATE VIRTUAL TABLE IF NOT EXISTS contacts_fts
  USING fts5(jid, display_name, content='contacts', content_rowid='rowid');

CREATE TRIGGER IF NOT EXISTS contacts_ai AFTER INSERT ON contacts BEGIN
  INSERT INTO contacts_fts(rowid, jid, display_name)
    VALUES (new.rowid, new.jid, new.display_name);
END;

CREATE TRIGGER IF NOT EXISTS contacts_ad AFTER DELETE ON contacts BEGIN
  INSERT INTO contacts_fts(contacts_fts, rowid, jid, display_name)
    VALUES ('delete', old.rowid, old.jid, old.display_name);
END;

CREATE TRIGGER IF NOT EXISTS contacts_au AFTER UPDATE ON contacts BEGIN
  INSERT INTO contacts_fts(contacts_fts, rowid, jid, display_name)
    VALUES ('delete', old.rowid, old.jid, old.display_name);
  INSERT INTO contacts_fts(rowid, jid, display_name)
    VALUES (new.rowid, new.jid, new.display_name);
END;
```

Then modify `getContacts()`: when `search` is present, use a completely different SQL query that JOINs via `contacts_fts`:

```typescript
if (search) {
  // FTS5 MATCH — instant indexed search instead of LIKE full table scan.
  // Phase 13 (SYNC-02, SYNC-05). DO NOT revert to LIKE — FTS5 is indexed, LIKE is O(n).
  const ftsQuery = search.split(/\s+/).filter(Boolean).map(t => this._fts5Quote(t)).join(' ');
  const typeCond = type === "contact"
    ? "AND c.is_group = 0 AND c.jid NOT LIKE '%@newsletter'"
    : type === "group"
    ? "AND c.is_group = 1"
    : type === "newsletter"
    ? "AND c.jid LIKE '%@newsletter'"
    : "";
  const ftsSql = `
    SELECT c.jid, c.display_name, c.first_seen_at, c.last_message_at, c.message_count, c.is_group,
           d.mode, d.mention_only, d.custom_keywords, d.can_initiate, d.can_initiate_override
    FROM contacts_fts
    JOIN contacts c ON contacts_fts.rowid = c.rowid
    LEFT JOIN dm_settings d ON c.jid = d.jid
    WHERE contacts_fts MATCH ?
      AND c.jid NOT LIKE '%@lid'
      AND c.jid NOT LIKE '%@s.whatsapp.net'
      ${typeCond}
    ORDER BY rank
    LIMIT ? OFFSET ?
  `;
  const rows = this.db.prepare(ftsSql).all(ftsQuery, limit, offset) as Array<Record<string, unknown>>;
  return rows.map(this._rowToContact);
}
```

Place this FTS5 branch BEFORE the existing LIKE-based query so it returns early. The existing LIKE path remains as a fallback for when search is empty (normal pagination without search).

8. **triggerImmediateSync(accountId: string): void** — public function:
- Clears existing timer for this accountId from syncTimers
- Gets SyncState — if currently "running", do nothing (let current cycle finish)
- Immediately calls tick() for this accountId (reset timer chain)

9. **tick(opts: SyncOptions, state: SyncState): Promise<void>** — private, core loop:
- Check `opts.abortSignal.aborted`, return if true
- Set state.status = "running", record start time
- try: call `runSyncCycle(opts, state)`, set status "idle", update lastSyncAt/lastSyncDuration/itemsSynced/lastError=null
- catch: set status "error", lastError = error message, console.warn
- Schedule next tick via setTimeout (chain pattern), store timer in syncTimers
