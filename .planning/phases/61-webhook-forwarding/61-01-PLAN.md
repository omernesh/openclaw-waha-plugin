    **Forwarder module (src/webhook-forwarder.ts):**
    Create module with these exports:
    - `signWebhookPayload(body: string, secret: string): string` — returns `sha256=` + HMAC-SHA256 hex using `node:crypto` createHmac
    - `forwardWebhook(params: { subscriptions, secret, eventType, payload })` — iterates enabled subscriptions matching eventType, calls deliverWithRetry for each, fire-and-forget errors logged
    - Internal `deliverWithRetry(url, body, headers, opts?)` — POST with AbortController 5s timeout; retry on 5xx/network error with delays [1000, 2000, 4000]; dead-letter on 4xx; circuit breaker check before each attempt
    - Internal circuit breaker: `Map<string, CircuitState>` keyed by URL. `CircuitState = { consecutiveTimeouts: number, openedAt: number | null }`. Opens after 3 consecutive timeouts. Half-opens after 60s. Closes on successful delivery.
    - DI: accept `_fetch`, `_sleep`, `_now` for test injection (avoid mocking globals)
    - Use structured logger child: `log.child({ module: "webhook-forwarder" })`
    - Export `resetCircuitBreakers()` for test cleanup

    - Use vitest, mock fetch via DI `_fetch` parameter
    - Mock `_sleep` to resolve immediately (no real delays in tests)
    - Test all behaviors listed above
    - Use `resetCircuitBreakers()` in beforeEach
