1. Install SDK: `npm install @modelcontextprotocol/sdk@^1.28.0`

2. Create `src/mcp-server.ts` exporting `createMcpServer(cfg: CoreConfig)` that returns a configured `McpServer`.

3. Register these 10 tools (MCP-01) using `server.registerTool()` with Zod input schemas. Each tool delegates to existing business logic from api-v1.ts/proxy-send-handler.ts/send.ts/directory.ts:

   - **send_message**: args `{ chatId: string, session: string, text: string }` -> `handleProxySend({ body: { chatId, session, text, type: "text" }, cfg })`
   - **send_media**: args `{ chatId: string, session: string, url: string, type: "image"|"video"|"file"|"voice", caption?: string }` -> `handleProxySend({ body: { chatId, session, url, type, caption }, cfg })`
   - **read_messages**: args `{ chatId: string, session: string, limit?: number }` -> `getWahaChatMessages({ cfg, accountId: session, chatId, limit: limit ?? 20, downloadMedia: false })`
   - **search**: args `{ query: string, limit?: number }` -> `db.getContacts({ search: query, limit })` for contacts + groups. Use first enabled account's directory DB.
   - **get_directory**: args `{ type?: "contact"|"group"|"newsletter", limit?: number, offset?: number, search?: string }` -> `db.getContacts(...)` same as api-v1.ts directory route
   - **manage_group**: args `{ action: "create"|"rename"|"add_participant"|"remove_participant"|"promote_admin"|"demote_admin"|"leave"|"info", session: string, groupId?: string, name?: string, participants?: string[] }` -> dispatch to `callWahaApi(...)` with appropriate WAHA endpoint per action
   - **get_status**: args `{ session?: string }` -> list sessions with health + mimicry cap status, same logic as api-v1.ts sessions + mimicry routes combined
   - **update_settings**: args `{ path: string, value: unknown }` -> `modifyConfig(configPath, cfg => set(cfg, path, value))`. Use lodash-style dot-path setter or simple implementation. Only allow paths under `channels.waha`.
   - **send_poll**: args `{ chatId: string, session: string, name: string, options: string[], multipleAnswers?: boolean }` -> `handleProxySend({ body: { chatId, session, type: "poll", poll: { name, options, multipleAnswers } }, cfg })`
   - **send_reaction**: args `{ chatId: string, session: string, messageId: string, reaction: string }` -> `callWahaApi({ path: "/api/sendReaction", method: "PUT", body: { chatId, messageId, reaction }, session, cfg })`

4. Register 5 resources (MCP-04) using `server.registerResource()`:

   - **chatlytics://sessions** (fixed): List enabled sessions with health state
   - **chatlytics://contacts/{jid}** (template via ResourceTemplate): Single contact details from directory DB
   - **chatlytics://groups/{jid}** (template via ResourceTemplate): Single group details from directory DB
   - **chatlytics://config** (fixed): Current waha channel config (sanitized — no API keys)
   - **chatlytics://mimicry** (fixed): Per-session mimicry cap status (read-only via getCapStatus, NEVER checkAndConsumeCap)

5. Error handling (MCP-05): Create a `buildRecoveryHint(body: Record<string, unknown>)` helper. For every tool callback:
   - If `handleProxySend` returns `body.blocked === true`: return `{ content: [{ type: "text", text: "Blocked: ${body.error}\n\nRecovery: ${hint}" }], isError: true }`
   - If result status >= 400: return `{ content: [{ type: "text", text: String(body.error) }], isError: true }`
   - Recovery hints: gate closed -> "Gate closed until HH:MM — retry then or use update_settings to adjust gate hours"; cap exceeded -> "Hourly cap of N reached — wait for window reset or increase cap via update_settings"; unknown -> "Check get_status for current gate/cap state"
   - Wrap every tool callback in try/catch; caught errors return `isError: true` with the error message, never throw.

6. McpServer constructor: `new McpServer({ name: "chatlytics", version: "2.0.0" })`
