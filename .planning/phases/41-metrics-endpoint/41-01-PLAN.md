# Plan 41-01: Prometheus Metrics Endpoint

## Objective

Create a `/metrics` endpoint exposing process-level metrics in Prometheus text format. No external libraries -- hand-format the exposition text.

## Context

- @src/monitor.ts — HTTP server, route placement
- @src/http-client.ts — outbound API call stats
- @src/inbound-queue.ts — queue depth/stats
- @src/health.ts — session health states
- @src/analytics.ts — event counts

## Tasks

### Task 1: Create src/metrics.ts module
type="auto"

Create `src/metrics.ts` with:
- Counters/gauges stored as module-level state
- `collectMetrics()` function returning Prometheus text format string
- Metrics to expose:
  - `process_heap_used_bytes` (gauge) — from process.memoryUsage()
  - `process_heap_total_bytes` (gauge)
  - `process_rss_bytes` (gauge)
  - `nodejs_eventloop_lag_seconds` (gauge) — measured via setTimeout(0) delta
  - `waha_http_requests_total` (counter, labels: route, method, status) — admin API requests
  - `waha_http_request_duration_seconds` (histogram, buckets: 0.01, 0.05, 0.1, 0.5, 1, 5) — admin API latency
  - `waha_inbound_queue_depth` (gauge, labels: priority) — DM and group queue depths
  - `waha_inbound_processed_total` (counter)
  - `waha_inbound_errors_total` (counter)
  - `waha_api_calls_total` (counter, labels: method, status) — outbound WAHA API calls
  - `waha_api_errors_total` (counter)
  - `waha_session_health` (gauge, labels: session, status) — 1 for current status, 0 for others
- Export `recordHttpRequest(route, method, status, durationMs)` for monitor.ts to call
- Export `recordApiCall(method, success)` for http-client.ts to call
- Export `updateQueueStats(stats: QueueStats)` for inbound-queue callback
- Export `updateSessionHealth(session, status)` for health callback

**Done when:** `collectMetrics()` returns valid Prometheus text format string with all metric families.

### Task 2: Wire metrics into monitor.ts and http-client.ts
type="auto"

1. Add `/metrics` route in monitor.ts BEFORE the admin auth check:
   ```
   if (req.url === "/metrics" && req.method === "GET") {
     res.writeHead(200, { "Content-Type": "text/plain; version=0.0.4; charset=utf-8" });
     res.end(collectMetrics());
     return;
   }
   ```
2. Wrap admin API route handling with timing + call `recordHttpRequest()` after response
3. In http-client.ts: call `recordApiCall()` after each fetch completes
4. Wire queue stats callback to `updateQueueStats()`
5. Wire health state callback to `updateSessionHealth()`

**Done when:** `/metrics` endpoint returns real data; TypeScript compiles clean.
