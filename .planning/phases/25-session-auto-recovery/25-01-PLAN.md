8. **Add `attemptRecovery` async function** (private):
```typescript
async function attemptRecovery(opts: HealthCheckOptions, state: HealthState): Promise<void>
```
Logic:
- Get or create RecoveryState from sessionRecoveryStates map
- Check cooldown: if `Date.now() < recoveryState.cooldownUntil`, log `[WAHA] Recovery skipped for session ${opts.session} — cooldown until ${new Date(cooldownUntil).toISOString()}` and return
- Set `recoveryState.cooldownUntil = Date.now() + RECOVERY_COOLDOWN_MS`
- Increment `recoveryState.attemptCount`
- Set `recoveryState.lastAttemptAt = Date.now()`
- Log: `[WAHA] Attempting auto-recovery for session ${opts.session} (attempt #${attemptCount}, ${state.consecutiveFailures} consecutive failures)`
- Try `callWahaApi({ baseUrl: opts.baseUrl, apiKey: opts.apiKey, path: \`/api/sessions/${opts.session}/restart\`, method: "POST", timeoutMs: 30_000, context: { action: "session-recovery" } })`
- On success: set `lastOutcome = "success"`, `lastError = null`, log `[WAHA] Auto-recovery SUCCESS for session ${opts.session}`
- On failure: set `lastOutcome = "failed"`, `lastError = errMsg`, log `[WAHA] Auto-recovery FAILED for session ${opts.session}: ${errMsg}`
- Push RecoveryEvent to recoveryHistory (shift oldest if > RECOVERY_HISTORY_MAX)
- After recovery attempt (success or fail), call `alertGodModeUsers(opts, state, recoveryState)` — fire and forget with .catch(warn)

9. **Add `alertGodModeUsers` async function** (private):
```typescript
async function alertGodModeUsers(opts: HealthCheckOptions, healthState: HealthState, recoveryState: RecoveryState): Promise<void>
```
Logic:
- If `!opts.cfg` return (no config = can't alert)
- Import `listEnabledWahaAccounts` from `./accounts.js` and `sendWahaText` from `./send.js` (dynamic import to avoid circular deps)
- Find a healthy session to send from: iterate `listEnabledWahaAccounts(opts.cfg)`, find one where `getHealthState(s.session)?.status === "healthy"` and `s.session !== opts.session`
- If no healthy session found, log `[WAHA] Cannot alert god mode users — no healthy session available` and return
- Extract god mode users from BOTH dmFilter and groupFilter config: `const godUsers = [...(cfg.channels?.waha?.dmFilter?.godModeSuperUsers ?? []), ...(cfg.channels?.waha?.groupFilter?.godModeSuperUsers ?? [])]`
- Deduplicate by identifier: `const uniqueJids = [...new Set(godUsers.map(u => typeof u === 'string' ? u : u.identifier))]`
- If no god mode users, return silently
- Build alert message: `"[WAHA Alert] Session ${opts.session} is unhealthy (${healthState.consecutiveFailures} failures). Auto-recovery attempt #${recoveryState.attemptCount}: ${recoveryState.lastOutcome?.toUpperCase()}${recoveryState.lastError ? ' — ' + recoveryState.lastError : ''}"`
- For each unique JID, call `sendWahaText({ cfg: opts.cfg, to: jid, text: alertMessage, accountId: healthyAccountId, bypassPolicy: true })` — use bypassPolicy since this is a system alert. Wrap each in try/catch, log failures but don't throw.

10. **Wire recovery into the `tick` function:**

if (opts.enableRecovery && state.consecutiveFailures >= AUTO_RECOVERY_THRESHOLD) {
