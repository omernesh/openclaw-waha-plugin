**src/pairing.ts -- PairingEngine class:**

```typescript
import { createHash, createHmac, randomInt } from "node:crypto";
import { getDirectoryDb } from "./directory.js";

export class PairingEngine {
  private accountId: string;
  private hmacSecret: string;

  constructor(accountId: string, hmacSecret: string) { ... }

  // Generate 6-digit passcode, hash with SHA-256, store in pairing_challenges table
  // Returns { passcode: string (plaintext for display to admin), challengeId: string }
  createChallenge(jid: string): { passcode: string; challengeId: string }

  // Verify a passcode attempt against stored hash
  // Returns { success: boolean; reason?: "correct" | "wrong" | "locked" | "expired" | "not_found" }
  // On wrong: increment attempts, lock if attempts >= 3 (set locked_until = now + 30min)
  // On correct: delete challenge row, return success
  verifyPasscode(jid: string, attempt: string): { success: boolean; reason: string }

  // Generate HMAC-SHA256 token: hmac(jid, secret) truncated to 12 hex chars
  // Used for wa.me deep links: https://wa.me/{phone}?text=PAIR-{token}
  // JID-specific: anyone with the correct link for their JID gets auto-paired
  generateDeepLinkToken(jid: string): string

  // Verify HMAC token: recompute hmac(senderJid, secret) and compare
  // No DB lookup needed -- pure HMAC verification
  verifyDeepLinkToken(senderJid: string, token: string): boolean

  // Check if a JID has an active (non-expired, non-locked) challenge
  hasActiveChallenge(jid: string): boolean

  // Get active grants (allow_list entries with source='pairing')
  getActiveGrants(): Array<{ jid: string; expiresAt: number | null; grantedAt: number | null }>

  // Revoke a pairing grant (remove from allow_list where source='pairing')
  revokeGrant(jid: string): void
}
```

**Implementation details for PairingEngine:**
- Passcode: `randomInt(100000, 999999).toString()` -- always 6 digits
- Hash: `createHash('sha256').update(passcode).digest('hex')`
- HMAC token: `createHmac('sha256', this.hmacSecret).update(jid).digest('hex').slice(0, 12)` -- 12 hex chars is enough for obfuscation while keeping the deep link short
- Deep link verification: extract token from message text matching `PAIR-([a-f0-9]{12})`, recompute `hmac(senderJid, hmacSecret).slice(0, 12)` and compare with timing-safe equal
- Use `timingSafeEqual` from node:crypto for token comparison to prevent timing attacks
- Challenge expiry: 24 hours from creation (created_at + 86400 < now means expired)
- Lock duration: 30 minutes (locked_until stored as Unix seconds)
- Use `getDirectoryDb(this.accountId)` for all SQLite access
- When granting access on successful pairing, call `setContactAllowDmWithSource(jid, true, expiresAt, 'pairing')` where expiresAt = now + grantTtlMinutes * 60

**HMAC secret:** Auto-generated via `randomBytes(32).toString('hex')` on first use if config field `pairingMode.hmacSecret` is empty. Stored back to config.
