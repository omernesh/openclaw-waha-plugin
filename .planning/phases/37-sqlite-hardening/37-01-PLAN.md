# Phase 37, Plan 01: SQLite Hardening & Temp File Cleanup

## Objective

Add SQLite busy_timeout and periodic WAL checkpointing to both database classes, and sweep orphaned media temp files on startup.

## Context

- @src/directory.ts — DirectoryDb constructor (WAL mode already set)
- @src/analytics.ts — AnalyticsDb constructor (WAL mode already set)
- @src/monitor.ts — monitorWahaProvider (startup entry point)
- @src/media.ts — temp file path pattern: /tmp/openclaw/waha-media-*

## Tasks

### Task 1: SQLite busy_timeout + WAL checkpoint + temp file sweep `type="auto"`

**Changes:**

1. **MEM-03 — busy_timeout**: Add `this.db.pragma("busy_timeout = 5000")` after WAL mode pragma in both `DirectoryDb` and `AnalyticsDb` constructors.

2. **DI-01 — WAL checkpoint timer**: Add a `startWalCheckpoint()` method to both DB classes that runs `PRAGMA wal_checkpoint(PASSIVE)` every 30 minutes via a setTimeout chain. Call it from the constructor. Store the timer handle and clear it in `close()` (add close() to AnalyticsDb if missing).

3. **MEM-02 — temp file cleanup**: In `monitorWahaProvider` (monitor.ts), before `server.start()`, sweep `/tmp/openclaw/waha-media-*` files older than 10 minutes using fs/promises readdir + stat + unlink. Fire-and-forget (don't block startup). Log count of cleaned files.

**Verification:**
- `npx tsc --noEmit` compiles clean

**Done when:**
- busy_timeout pragma present in both DBs
- WAL checkpoint timer runs in both DBs
- Temp file sweep runs on startup
- TypeScript compiles

## Success Criteria

- [ ] WAL checkpoint timer (30 min) added to both DBs
