<feature>
  <name>enforceMimicry chokepoint + recordMimicrySuccess</name>
  <files>src/mimicry-enforcer.ts, src/send-pipeline.test.ts</files>
  <behavior>
    Test 1: bypassPolicy=true skips all checks — returns immediately, no gate/cap/delay
    Test 2: Time gate blocked — throws Error with message containing "[mimicry] Send blocked:" and "outside send window"
    Test 3: Cap exceeded — throws Error with message containing "[mimicry] Send blocked:" and "Hourly cap"
    Test 4: Jitter delay applied — sleep called with value between 3000 and 7000 (base 5000 +/-40%)
    Test 5: Typing indicator duration = Math.min(messageLength / 4 * 1000, 8000) ms; sendWahaPresence called with typing:true then typing:false
    Test 6: No messageLength (0 or undefined) — no typing indicator fired, only jitter delay
    Test 7: Batch pre-check with count=5, remaining cap=3 — throws cap exceeded error
    Test 8: Batch pre-check with count=3, remaining cap=5 — passes (does not throw)
    Test 9: recordMimicrySuccess calls db.recordSend(session)
    Test 10: Status sends (isStatusSend=true) skip cap check but still check time gate
  </behavior>
  <implementation>
    Create src/mimicry-enforcer.ts with:

    1. `enforceMimicry(params)` async function:
       - Params: `{ session: string; chatId: string; accountId: string; cfg: CoreConfig; bypassPolicy?: boolean; messageLength?: number; count?: number; isStatusSend?: boolean }`
       - Step 1: if bypassPolicy, return immediately
       - Step 2: Resolve per-target overrides from DirectoryDb (getDmSettings)
       - Step 3: checkTimeOfDay — throw on blocked with structured "[mimicry] Send blocked: {reason}" error
       - Step 4: if NOT isStatusSend, cap pre-check using db.countRecentSends + resolveCapLimit — throw if count + (opts.count ?? 1) > limit
       - Step 5: Jitter delay — base 5000ms, jitter = base * 0.4, actual = base + (Math.random() * 2 - 1) * jitter
       - Step 6: if messageLength > 0, typing simulation — sendWahaPresence(typing:true), sleep(min(msgLen/4*1000, 8000)), sendWahaPresence(typing:false)

    2. `recordMimicrySuccess(session: string)` function:
       - Calls getMimicryDb().recordSend(session)

    Dependencies injected via params (cfg, session, accountId) to avoid module-level state issues.
    Import from mimicry-gate.ts: checkTimeOfDay, resolveGateConfig, resolveCapLimit, getMimicryDb, getMaturityPhase
    Import from send.ts: sendWahaPresence
    Import from directory.ts: getDirectoryDb

    Test approach: mock sendWahaPresence, mock sleep (vi.fn for setTimeout/delay), use real MimicryDb with temp SQLite files.
  </implementation>
</feature>
