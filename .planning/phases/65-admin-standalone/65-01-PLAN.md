    **GET /api/admin/workspaces** — List all workspaces:
    - Query auth.db: `SELECT id, name, email, workspaceId, createdAt FROM user WHERE workspaceId IS NOT NULL ORDER BY createdAt DESC`
    - Cross-reference with WorkspaceProcessManager.listWorkspaces() to get runtime status (running/stopped/starting) and port

    - The `manager` (WorkspaceProcessManager) reference is available in the route handler scope — check how existing routes access it (likely via `opts` or closure)

    **POST /api/admin/workspaces** — Create a new workspace:
    - Parse JSON body: `{ name: string, email: string, password: string }`
    - Call `auth.api.signUpEmail({ body: { name, email, password } })` to create user (workspaceId auto-assigned by databaseHooks)
    - Return 201 with the created user record (including workspaceId)
    - Return 400 on validation errors, 409 if email already exists

    **DELETE /api/admin/workspaces/:workspaceId** — Delete a workspace:
    - Parse workspaceId from URL path (e.g., `req.url.match(/\/api\/admin\/workspaces\/([^/]+)$/)`)
    - Call `manager.stopWorkspace(workspaceId)` FIRST to stop the child process (Pitfall 3 from research)
    - Then delete from auth.db: `DELETE FROM user WHERE workspaceId = ?`
    - Return 200 on success, 404 if workspace not found
