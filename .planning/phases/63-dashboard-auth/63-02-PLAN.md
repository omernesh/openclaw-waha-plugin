**C. Add QR proxy route in monitor.ts:**
Inside the admin API section (after requireAdminAuth guard), add:
```typescript
// Phase 63 (AUTH-03): QR code proxy — fetches WAHA QR and session status.
// GET /api/admin/qr?session=xxx — returns { qrBase64, mimetype, status }
// Polls WAHA GET /api/{session}/auth/qr (Accept: application/json) for base64 QR
// Then GET /api/{session} for status (SCAN_QR_CODE | WORKING | FAILED)
// Also supports POST /api/admin/qr/start?session=xxx to create a new WAHA session
```
Implementation:
1. `GET /api/admin/qr` with `?session=` query param
2. Call `callWahaApi(cfg, "GET", \`/api/\${session}/auth/qr\`, null, { Accept: "application/json" })` to get QR
3. Call `callWahaApi(cfg, "GET", \`/api/\${session}\`)` to get session status
4. Return `{ qrBase64: data.data, mimetype: data.mimetype, status: sessionInfo.status }`
5. Handle errors gracefully — if session doesn't exist, return `{ error: "session_not_found" }`
6. `POST /api/admin/qr/start` — calls `callWahaApi(cfg, "POST", "/api/sessions/start", { name: session, config: { noweb: { store: { enabled: true } } } })` to provision a new WAHA session
