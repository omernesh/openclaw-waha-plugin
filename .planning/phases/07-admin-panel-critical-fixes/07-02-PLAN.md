**Root cause:** The GET /api/admin/directory handler (monitor.ts ~line 2115) filters out `@lid` and `@s.whatsapp.net` entries AFTER the SQL query runs with LIMIT/OFFSET. This means:
- If the DB has 50 contacts and 49 are `@lid`, only 1 shows (bug: "initial load shows only 1 contact")
- `total` from `getContactCount()` includes the filtered entries, so `dirOffset < d.total` is always true, causing Load More to keep fetching the same page
- The `dirOffset` on the client advances by the enriched (post-filter) count, but the server's SQL OFFSET needs the pre-filter count, causing duplicates

**Fix in src/directory.ts:**

1. In `getContacts()` method (~line 193), add a JID exclusion filter to the SQL WHERE clause. After the existing type filter conditions, always add:
   ```typescript
   conditions.push("c.jid NOT LIKE '%@lid' AND c.jid NOT LIKE '%@s.whatsapp.net'");
   ```
   This goes BEFORE the search filter condition block, right after the type filter block (~line 215). Place it as an always-applied condition (not inside an if block).

2. In `getContactCount()` method (~line 288), add the same exclusion. After the existing type conditions block, add:
   ```typescript
   conditions.push("jid NOT LIKE '%@lid' AND jid NOT LIKE '%@s.whatsapp.net'");
   ```
   Note: `getContactCount` uses `jid` directly (no table alias), while `getContacts` uses `c.jid`.
