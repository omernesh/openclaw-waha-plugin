    2. Create `Dockerfile` (multi-stage):
       ```dockerfile
       # Stage 1: Build admin panel (Vite)
       FROM node:22-slim AS builder
       WORKDIR /app
       COPY package*.json ./
       RUN npm ci
       COPY . .
       RUN npm run build:admin

       # Stage 2: Runtime
       FROM node:22-slim
       RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/*
       WORKDIR /app

       COPY package*.json ./
       RUN npm ci --omit=dev && npm install tsx

       COPY src/ ./src/
       COPY index.ts ./
       COPY SKILL.md ./
       COPY config-example.json ./
       COPY --from=builder /app/dist/admin/ ./dist/admin/

       # Create default data directory
       RUN mkdir -p /data && chown node:node /data

       USER node

       ENV NODE_ENV=production
       ENV CHATLYTICS_DATA_DIR=/data
       EXPOSE 8050

       HEALTHCHECK --interval=10s --timeout=5s --retries=6 --start-period=30s \
         CMD curl -f http://localhost:8050/health || exit 1

       CMD ["npx", "tsx", "src/standalone.ts"]
       ```

       Key decisions:
       - `node:22-slim` (not alpine) because better-sqlite3 native binding needs glibc
       - `curl` installed for HEALTHCHECK (node:22-slim doesn't include it)
       - `npm ci --omit=dev` keeps image lean, then `npm install tsx` adds only the runner
       - `USER node` for security (non-root)
       - `CHATLYTICS_DATA_DIR=/data` — Docker volume mounts here
       - Port 8050 matches the existing monitor.ts default

    3. Create `docker-compose.yml`:
       ```yaml
       services:
         chatlytics:
           build: .
           ports:
             - "${CHATLYTICS_PORT:-8050}:8050"
           environment:
             - CHATLYTICS_CONFIG_PATH=/config/config.json
             - CHATLYTICS_DATA_DIR=/data
           volumes:
             - ./config.json:/config/config.json:ro
             - chatlytics-data:/data
           restart: unless-stopped

       volumes:
         chatlytics-data:
       ```

       Notes:
       - Config file bind-mounted read-only from host ./config.json
       - Named volume `chatlytics-data` persists SQLite databases across restarts
       - `CHATLYTICS_PORT` env var on host controls port binding (default 8050)
       - No healthcheck in compose — Dockerfile HEALTHCHECK is authoritative
       - `restart: unless-stopped` for production resilience
  </action>
