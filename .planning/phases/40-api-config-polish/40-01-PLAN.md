# Phase 40 Plan 01: API & Config Polish

## Objective

Fix four low-hanging code quality issues: admin API rate limiting, req.url mutation, config schema bounds, and configureReliability race condition.

## Context

- @src/monitor.ts — admin route handler, req.url mutation on line 613
- @src/http-client.ts — configureReliability() overwrites global singleton per-account
- @src/config-schema.ts — healthCheckIntervalMs lacks .min(), syncIntervalMinutes already has .min(0)
- @src/rate-limiter.ts — RateLimiter class available for reuse

## Tasks

### Task 1: Admin API rate limiting (API-01)
type="auto"

Add a simple in-memory rate limiter for `/api/admin/*` routes in monitor.ts. Use a sliding window counter (Map<ip, {count, resetAt}>) — simpler than importing RateLimiter which is concurrency-based, not request-count-based. Limit: 60 requests per minute per IP. Return 429 with JSON `{ error: "Too many requests", retryAfterMs }` when exceeded.

**Done when:** Admin routes return 429 after exceeding limit. Non-admin routes (webhooks, health) unaffected.

### Task 2: Fix req.url mutation (API-02)
type="auto"

In monitor.ts line 613, replace `req.url = req.url.slice("/admin".length)` with a local variable. The subsequent code on line 614 uses `req.url` so we need to use the local variable there too.

**Done when:** `req.url` is never mutated. Asset serving still works correctly.

### Task 3: Config schema bounds (CFG-01)
type="auto"

In config-schema.ts, add `.min(10000)` to `healthCheckIntervalMs` to prevent dangerously fast health checks. `syncIntervalMinutes` already has `.min(0)` which is correct (0 = disabled).

**Done when:** Setting healthCheckIntervalMs below 10000 fails Zod validation.

### Task 4: Fix configureReliability race (CFG-02)
type="auto"

In http-client.ts, change from a single global bucket to a per-account bucket map. `configureReliability()` takes an accountId and stores the bucket. `callWahaApi()` already receives context but needs an accountId field to look up the right bucket. Add `accountId?: string` to `CallWahaApiParams` — if absent, use a default bucket (backward compat).

**Done when:** Multiple accounts each get their own TokenBucket. Last-account-wins eliminated.

## Verification

- `npx tsc --noEmit` compiles clean
- All four changes in place
