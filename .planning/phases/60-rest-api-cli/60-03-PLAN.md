    - Test: --api-key flag is sent as Authorization Bearer header

    - Test: --url flag overrides default base URL

    - Test: --json flag outputs raw JSON (no table formatting)
    - Test: without --json, output includes table borders or chalk formatting
  </behavior>
  <action>
    1. Create `src/cli.ts` with shebang `#!/usr/bin/env node`:

    ```typescript
    import { Command } from 'commander';
    import chalk from 'chalk';
    import Table from 'cli-table3';
    ```

    2. Define helper function `apiCall(path, opts, method='GET', body?)`:
       - Builds URL from `opts.parent?.opts().url ?? 'http://localhost:8050'` + path
       - Sets `Authorization: Bearer {apiKey}` header if apiKey provided
       - apiKey from `opts.parent?.opts().apiKey ?? process.env.CHATLYTICS_API_KEY`
       - Returns parsed JSON response
       - On non-2xx, throws with error message from response body

    3. Define `outputResult(data, opts, formatFn)`:
       - If `opts.parent?.opts().json` is true, print `JSON.stringify(data, null, 2)`
       - Otherwise call `formatFn(data)` which uses chalk + cli-table3

    4. Define commands:

       **send** `<message>`:
       - Options: `--to <target>` (required), `--session <session>` (optional)
       - POST /api/v1/send with body `{ text: message, chatId: opts.to, session: opts.session }`
       - Table output: single row with messageId, status
       - Success: green "Message sent" + messageId

       **search** `<query>`:
       - GET /api/v1/search?q={query}
       - Table output: columns [Type, Name, JID] with contacts and groups

       **status**:
       - GET /api/v1/sessions
       - Table output: columns [Session, Name, Status, Webhook] with color (green=connected, red=disconnected)

       **messages**:
       - Options: `--chat <chatId>` (required), `--session <session>` (required), `--limit <n>` (default 20)
       - GET /api/v1/messages?chatId={chat}&session={session}&limit={limit}
       - Table output: columns [Time, From, Message] with truncated text

       **directory**:
       - Options: `--type <type>` (optional: contact|group|newsletter), `--search <term>`, `--limit <n>`, `--offset <n>`
       - GET /api/v1/directory with query params
       - Table output: columns [Type, Name, JID]

       **mimicry**:
       - GET /api/v1/mimicry
       - Table output: columns [Session, Cap Used, Cap Limit, Gate Open] with color

    5. Global options on the program:
       - `--url <url>` with default from `process.env.CHATLYTICS_URL ?? 'http://localhost:8050'`
       - `--api-key <key>` with default from `process.env.CHATLYTICS_API_KEY`
       - `--json` boolean flag

    6. End with `program.parseAsync(process.argv)`.

    7. Write `tests/cli.test.ts`:
       - Mock global `fetch` to intercept HTTP calls
       - Test each subcommand sends correct HTTP request (URL, method, headers, body)
       - Test --json flag produces JSON.parse-able output
       - Test env var fallbacks for URL and API key
       - Test default URL is http://localhost:8050

    Per CLI-01: send, search, status, messages, directory, mimicry subcommands.
    Per CLI-02: CHATLYTICS_API_KEY env var or --api-key flag.
    Per CLI-03: CHATLYTICS_URL env var or --url flag, default http://localhost:8050.
    Per CLI-04: chalk + cli-table3 for human output, --json for machine output.
