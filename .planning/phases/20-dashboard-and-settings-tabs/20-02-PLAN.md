**Data fetching:**
- `useEffect` with `[refreshKey]` dependency: `api.getConfig()` to load current config
- AbortController for cleanup
- State: `config: WahaConfig | null`, `loading: boolean`, `saving: boolean`, `restarting: boolean`, `dirty: boolean`
- On successful load, spread config into form state. Track `dirty` flag — set to true on any field change, false after save.

**JID resolution for display:**
- On config load, collect all JIDs from: `config.allowFrom`, `config.groupAllowFrom`, `config.allowedGroups`, `config.dmFilter.godModeSuperUsers[].identifier`, `config.groupFilter.godModeSuperUsers[].identifier`
- Batch resolve via `api.resolveNames(allJids)` — store in `resolvedNames: Record<string, string>` state
- Pass to TagInput components as `resolvedNames` prop

**Directory search function (for JID TagInputs):**
```typescript
async function searchDirectory(query: string): Promise<Array<{ value: string; label: string }>> {
  const result = await api.getDirectory({ search: query, limit: '10' })
  return result.items.map(item => ({ value: item.jid, label: item.name || item.jid }))
}
```
Pass this as `searchFn` to JID-type TagInputs.

**Save buttons (sticky footer or bottom of form):**

Two buttons side by side:
1. **Save** — `api.updateConfig({ waha: buildPayload() })`. Shows saving spinner. On success, set dirty=false.
2. **Save & Restart** — same save, then triggers RestartOverlay.

**buildPayload() function — CRITICAL:**
Must construct the `waha` object with complete sub-objects (not partial). Per RESEARCH.md Pitfall 5, `deepMerge` on the server preserves keys not present in incoming. So:
- For simple fields: include current value
- For sub-objects (dmFilter, groupFilter, presence, etc.): send the complete sub-object
- CRITICAL: `godModeSuperUsers` must be serialized as `Array<{ identifier: string }>`, NOT `string[]`. Use: `godModeSuperUsers: jids.map(jid => ({ identifier: jid }))`
- Empty arrays MUST be sent explicitly as `[]` (not omitted) when user clears all values — otherwise server preserves old values

**RestartOverlay integration:**
- State: `restarting: boolean` controls overlay visibility
- Save & Restart flow: `await api.updateConfig(payload)` -> `setSaving(false)` -> `setRestarting(true)` -> RestartOverlay appears
- RestartOverlay `onComplete`: `window.location.reload()` (same as existing panel behavior)
- RestartOverlay `onTimeout`: `setRestarting(false)`, show error toast or alert

**Loading state:**
- While loading, show centered "Loading..." text
