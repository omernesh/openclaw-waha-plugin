**UI-03 — Optimistic UI for role save (BUG-04):**
In the `saveSessionRole()` function (client-side JS), change the save flow:
1. BEFORE the fetch call, capture the new dropdown values
2. Do NOT call `loadSessions()` after a successful save — this is what causes the flicker (full re-render resets to running config values)
3. On success: show toast "Saved. Restart required for changes to take effect." and leave dropdowns at their new values (they already show the correct value since the user just changed them)
4. On error: revert the dropdown to the previous value and show error toast

Currently the flow is likely: user changes dropdown -> saveSessionRole() fires -> fetch PUT -> on success call loadSessions() -> loadSessions() re-renders with the RUNNING config (not saved config) -> dropdown appears to "revert". The fix is to simply NOT call loadSessions() on success.

**UI-04 — 502 polling overlay (BUG-05):**
Find the existing restart polling overlay from the Settings tab (search for "Gateway restarting" or "restart" overlay pattern in the embedded JS). It likely creates an overlay div with "Gateway restarting..." text and polls an endpoint every 2 seconds until success.

Wrap the `saveSessionRole()` fetch call in error handling that specifically detects 502 status:
```javascript
fetch(url, opts).then(function(r) {
  if (r.status === 502) {
    // Show polling overlay — reuse or copy the Settings tab pattern
    showRestartOverlay();
    return;
  }
  if (!r.ok) throw new Error("Status " + r.status);
  // success handling...
}).catch(function(err) {
  // Also catch network errors (which happen during restart)
  showRestartOverlay();
});
```

The `showRestartOverlay()` function should:
- Create a full-screen semi-transparent overlay with "Gateway restarting..." text
- Poll `GET /api/admin/sessions` every 2 seconds
- On successful response (200), remove overlay and call `loadSessions()` to refresh
- Auto-dismiss after 30 seconds with error message if still failing

If the Settings tab already has this function, reuse it. If it is scoped inside a specific tab's code, extract it to a shared function accessible from both tabs.
