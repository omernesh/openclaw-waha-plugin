**UI-01 — Fix Access Control card flickering (BUG-02):**
Find what causes periodic re-rendering of the Access Control card. Look for `setInterval` or `setTimeout` calls that invoke `loadStats()` repeatedly. Either:
- Remove the periodic refresh entirely (dashboard is not a live monitoring screen), OR
- If periodic refresh is needed for health, isolate it so it only updates health data without rebuilding the entire dashboard DOM. The Access Control card must NOT re-render on each tick.

Key: The Name Resolver (`createNameResolver()`) fires async fetches per JID. If `loadStats()` re-runs, it rebuilds the Name Resolver divs, causing new fetches and visual flicker. The fix must ensure the Name Resolver divs are created ONCE and not destroyed on subsequent refreshes.
