  <action>
    Create src/workspace-gateway.ts:

    1. **API key cache**: `LRUCache<string, string>` with max 500 entries, TTL 60s. Key = bearer token, value = workspaceId.

    2. **resolveWorkspaceFromKey(bearerToken: string): Promise<string | null>** — check cache first. On miss, call `auth.api.verifyApiKey({ body: { key: bearerToken } })`. If result.valid and result.user.workspaceId, cache and return. Else return null.

    3. **proxyToWorkspace(req, res, port): Promise<void>** — use `node:http.request` to proxy. Set target host to 127.0.0.1, port from param, path from req.url, method from req.method, headers from req.headers. Pipe `proxyRes` to `res` with `res.writeHead(proxyRes.statusCode, proxyRes.headers)`. On error, return 502 `{ error: "Workspace unavailable" }`.

    4. **WorkspaceGateway class**:
       - Constructor takes `{ manager: WorkspaceProcessManager, port: number, host: string }`.
       - Creates `node:http` server with request handler that routes:
         a. `GET /healthz` — return 200 `{ status: "ok", workspaces: manager.listWorkspaces() }` (parent health)
         b. `/api/auth/*` — delegate to better-auth `toNodeHandler(auth.handler)` (parent handles auth)
         c. `/webhook/waha` — call handleWebhook (read body, extract session, route to child)
         d. `/api/v1/*` — extract Bearer token from Authorization header, resolveWorkspaceFromKey, get port from manager, proxyToWorkspace. Return 401 if key invalid, 503 if workspace crashed.
         e. `/` and static files — serve admin panel from ADMIN_DIST (same pattern as monitor.ts lines 87-97)
         f. `/mcp` — same as /api/v1/* routing (API key -> workspace -> proxy)
         g. Everything else — 404
       - `start(): Promise<void>` — listen on configured port/host
       - `stop(): Promise<void>` — close server, call manager.stopAll()

    5. **handleWebhook logic**: Read request body (reuse readWahaWebhookBody from monitor.ts), parse JSON, extract `body.session` string. Call `extractWorkspaceIdFromSession(session)` to get workspaceId. Get port from manager. If no workspaceId or no port, return 404. Otherwise pipe the original body to the child's /webhook/waha endpoint.

    6. **CORS**: Apply same CORS headers as api-v1-auth.ts for /api/v1/* routes.

    7. Add DO NOT CHANGE comments on: routing order (healthz before auth before webhook before api), the 401/503 distinction, the webhook session extraction logic.

    - Mock WorkspaceProcessManager (getPort, getStatus, listWorkspaces)
    - Mock auth.api.verifyApiKey
    - Test all behaviors listed above
    - Use `vi.fn()` for mocks, supertest-style or raw http.request for HTTP assertions
