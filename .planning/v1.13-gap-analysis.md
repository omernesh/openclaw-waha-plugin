## 4. Proposed Phases for v1.13

### Phase 25: Session Auto-Recovery (HIGH)

**Goal**: When health monitor detects unhealthy session, automatically attempt recovery before operator intervention.

**Tasks**:
- Add WAHA session restart API call on unhealthy threshold (e.g., after 5 consecutive failures)
- Add cooldown between restart attempts (e.g., 5 minutes)
- Log recovery attempts and outcomes
- Surface recovery events in admin panel Dashboard health cards
- Send alert to god mode users via healthy session when session goes unhealthy

**Complexity**: Medium
**Dependencies**: None

---

### Phase 26: Config Validation & Safety (HIGH)

**Goal**: Prevent config corruption from admin panel saves and provide config backup/restore.

**Tasks**:
- Validate POST /api/admin/config payload against Zod schema before saving
- Return validation errors as structured response (field-level errors)
- Surface validation errors in admin panel Settings tab (per-field error messages)
- Add config export endpoint (GET /api/admin/config/export — full JSON download)
- Add config import endpoint (POST /api/admin/config/import — with validation)
- Add config backup before save (rotate last 3 backups)

**Complexity**: Medium
**Dependencies**: None

---

### Phase 27: Real-Time Admin Panel (MEDIUM)

**Goal**: Admin panel receives live updates without manual refresh.

**Tasks**:
- Add SSE endpoint (GET /api/admin/events) for real-time push
- Emit events: health state changes, queue depth changes, new messages processed, config saves
- Add SSE client hook in React admin panel (useEventSource)
- Dashboard cards auto-update on health/queue events
- Log tab auto-scrolls on new entries
- Connection indicator in sidebar (connected/reconnecting)

**Complexity**: Large
**Dependencies**: None

---

### Phase 28: Conversation Analytics (MEDIUM)

**Goal**: Track and display message analytics for operational visibility.

**Tasks**:
- Add SQLite analytics table (message_events: timestamp, direction, chat_type, action, duration_ms, status)
- Record inbound/outbound events with processing time
- Add analytics API endpoint (GET /api/admin/analytics?range=24h&groupBy=hour)
- Add Analytics tab to admin panel with charts (messages/hour, response time distribution, top active chats)
- Use lightweight chart library (recharts — already in shadcn ecosystem)

**Complexity**: Large
**Dependencies**: None

---

### Phase 29: Test Coverage Sprint (MEDIUM)

**Goal**: Bring test coverage to critical paths that are currently untested.

**Tasks**:
- Add monitor.ts admin API route tests (mock HTTP req/res, test each endpoint)
- Add inbound.ts pipeline tests (mock SDK imports, test filter/dedup/queue flow)
- Add directory.ts CRUD tests (create/update/delete contacts, participants, overrides)
- Add shutup.ts interactive flow tests
- Add React component tests for admin panel tabs (vitest + testing-library)
- Target: every exported function has at least one test

**Complexity**: Large
**Dependencies**: None

---

### Phase 30: Operational Polish (LOW)

**Goal**: Quality-of-life improvements for operators.

**Tasks**:
- Log tab: add download/export button (CSV or plain text)
- Add audit trail table (config_changes: timestamp, field, old_value, new_value, source)
- Surface audit trail in admin panel (Settings tab history section)
- Fix `shutup.ts:239` remaining `.catch(() => {})`
- Resolve `inbound.ts:704` TODO (admin name resolution)
- Add `prefers-color-scheme` auto-detect to theme toggle

**Complexity**: Medium
**Dependencies**: None

## 5. Recommended v1.13 Scope

Based on value and risk, recommended phases for v1.13:

| Phase | Priority | Justification |
|-------|----------|---------------|
| **25: Session Auto-Recovery** | HIGH | #1 operational pain point — silent disconnects cause missed messages with no automatic recovery |
| **26: Config Validation & Safety** | HIGH | Config corruption risk from admin panel is real. Backup/restore is table-stakes for production use |
| **29: Test Coverage Sprint** | MEDIUM | 1980-line monitor.ts and 1100-line inbound.ts with zero tests is fragile. Tests prevent regressions (user's #1 frustration) |

**Defer to v1.14**: Phases 27 (real-time), 28 (analytics), 30 (polish). These are nice-to-have but don't address operational risks.
