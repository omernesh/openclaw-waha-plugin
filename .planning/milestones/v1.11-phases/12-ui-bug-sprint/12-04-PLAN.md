**UI-07 — Contact settings drawer stays open after save (BUG-09):**

The issue is that after saving contact settings, the drawer/panel collapses. This happens because the save success handler likely calls `loadDirectory()` which rebuilds the entire directory list, destroying the expanded card.

Fix approach:
1. Find the contact card save button click handler. It likely:
   a. Collects field values from the card
   b. Sends PUT to `/api/admin/directory/:jid/settings`
   c. On success: calls `loadDirectory()` or collapses the card

2. Change the success handler to NOT call `loadDirectory()` or collapse the card. Instead:
   a. Show a success toast: `showToast("Settings saved", false)`
   b. Leave the card expanded and all field values in place
   c. Optionally, update any displayed values that might have changed (e.g., a badge reflecting the new mode)

3. If the card is a `<details>` element, ensure the `open` attribute is preserved.

4. If the save handler removes and re-creates the card row to update a visual indicator (like mode badge), instead of destroying the card, update ONLY the badge text/color inline:
```javascript
// Instead of rebuilding the whole row:
var modeBadge = document.getElementById("mode-badge-" + sanitizedJid);
if (modeBadge) modeBadge.textContent = newMode;
```

5. If `loadDirectory()` must be called (e.g., to update list-level indicators), save the expanded state before calling it and restore after:
```javascript
var expandedJid = currentlyExpandedJid; // track which card is open
loadDirectory().then(function() {
  if (expandedJid) reopenContactCard(expandedJid);
});
```
However, this is the LESS preferred approach — better to not call loadDirectory() at all on save.
