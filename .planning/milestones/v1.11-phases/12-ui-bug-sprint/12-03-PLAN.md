2. **Create the wrapRefreshButton helper** in the embedded `<script>` section:
```javascript
function wrapRefreshButton(btn, loadFn) {
  var tsEl = document.createElement("span");
  tsEl.className = "refresh-ts";
  tsEl.textContent = "";
  btn.parentNode.insertBefore(tsEl, btn.nextSibling);

  var lastRefreshed = null;

  function updateTs() {
    if (!lastRefreshed) return;
    var diff = Math.floor((Date.now() - lastRefreshed) / 1000);
    if (diff < 60) tsEl.textContent = "Just now";
    else if (diff < 3600) tsEl.textContent = Math.floor(diff / 60) + "m ago";
    else tsEl.textContent = Math.floor(diff / 3600) + "h ago";
  }

  // Update relative time every 30s
  setInterval(updateTs, 30000);

  btn.addEventListener("click", function() {
    var origText = btn.textContent;
    btn.textContent = "Refreshing...";
    btn.classList.add("refreshing");
    btn.disabled = true;

    Promise.resolve(loadFn()).then(function() {
      lastRefreshed = Date.now();
      updateTs();
    }).catch(function(err) {
      console.warn("[waha] refresh failed:", err);
    }).then(function() {
      btn.textContent = origText;
      btn.classList.remove("refreshing");
      btn.disabled = false;
    });
  });
}
```

**UI-06 — Tooltip overflow fix (BUG-08):**
The `.tip::after` CSS pseudo-element renders tooltips, but they get clipped by parent containers with `overflow:hidden` or `overflow:auto`.

Fix approach:
1. Find the `.tip::after` CSS rule in the style block. Add or increase `z-index` to at least `1000`.
2. Find ALL parent containers of tooltip elements that have `overflow:hidden` or `overflow:auto`. These are likely `.card`, `.contact-card`, `.settings-drawer`, or similar card/panel classes.
3. Change those containers to `overflow:visible` — BUT only if this does not break scrollable content. If a container MUST scroll, use an alternative approach:
   - Set `position:relative` on the tooltip parent
   - Use a JS-based tooltip that creates a `position:fixed` element at the cursor position instead of CSS pseudo-elements

Simpler fix (preferred): Add `overflow:visible` to the direct parent of tooltip elements. The tooltip's `.tip` span is inside form fields which are inside a contact card panel. The panel div likely has `overflow:hidden` or `overflow-y:auto`. Change the panel to `overflow:visible` and if scrolling is needed, apply scroll only to the inner content area, not the panel that contains tooltips.
