  <name>Task 1: Rules resolver for DM and group flows</name>
  <files>src/rules-resolver.ts, tests/rules-resolver.test.ts</files>
  <behavior>
    - resolveContactPolicy(chatId, basePath): loads global default + specific override, merges, returns merged ContactRule
    - resolveContactPolicy with no override file: returns global default only
    - resolveContactPolicy with malformed override: returns global default (override skipped)
    - resolveGroupPolicy(chatId, senderId, basePath): loads group default + override, merges, evaluates contact_rule_mode
    - resolveGroupPolicy with contact_rule_mode="ignore": does NOT load speaker contact policy
    - resolveGroupPolicy with contact_rule_mode="apply": loads and merges speaker contact policy
    - resolveGroupPolicy with contact_rule_mode="restricted": loads speaker contact policy but only uses trust_level and forbidden_actions from it
    - resolveGroupPolicy with participants_allowlist mode="everyone": speaker_allowed=true
    - resolveGroupPolicy with participants_allowlist mode="none": speaker_allowed=false
    - resolveGroupPolicy with participants_allowlist mode="explicit" and speaker in IDs: speaker_allowed=true
    - resolveGroupPolicy with participants_allowlist mode="explicit" and speaker NOT in IDs: evaluates unknown_participant_policy
    - resolveInboundPolicy({isGroup, chatId, senderId, basePath}): dispatches to contact or group resolver
    - resolveOutboundPolicy({chatId, basePath}): resolves policy for outbound target (DM or group)
    - Cache: second call with same scope+mtime returns cached result (verified by checking no second file read)
  </behavior>
  <action>
    1. Create `src/rules-resolver.ts`:
       - `resolveContactPolicy(params: { chatId: string; basePath: string; safeName?: string }): ResolvedPolicy` — Flow A from resolver-algorithm.md:
         a. Normalize chatId with normalizeToStableId.
         b. Check policyCache.get(stableId, mtime). If hit, return cached.
         c. Load global default: loadDefaultContactRule(basePath).
         d. Check for override file: use findOverrideFile, check if file exists with fs.existsSync.
         e. If override exists: loadContactRule(overridePath). If valid, merge with mergeRuleLayers([globalDefault, override]).
         f. Build ResolvedPolicy using buildDmPayload (from resolved-payload-builder.ts).
         g. Cache result with policyCache.set.
         h. Return resolved policy.
         i. Get file mtime with fs.statSync(path).mtimeMs for cache key. For default file, use the default file's mtime.
       - `resolveGroupPolicy(params: { chatId: string; senderId: string; basePath: string; safeName?: string; senderSafeName?: string }): ResolvedPolicy` — Flow B from resolver-algorithm.md:
         a. Normalize chatId and senderId with normalizeToStableId.
         b. Check policyCache for group scope. If hit, build payload and return.
         c. Load global group default + optional group override, merge.
         d. Evaluate participants_allowlist.mode:
            - "everyone" -> speaker_allowed = true
            - "none" -> speaker_allowed = false
            - "explicit" -> check if senderId is in participants_allowlist.ids (compare normalized IDs)
            - "admins" -> for v1, treat as "none" (admin list not available without WAHA call)
         e. If speaker not in allowlist, evaluate unknown_participant_policy:
            - "deny" -> speaker_allowed = false
            - "observe_only" -> speaker_allowed = true but participation_mode forced to "silent_observer"
            - "fallback_to_global_contact" -> load speaker's global contact default, use trust_level to decide
         f. Evaluate contact_rule_mode:
            - "ignore" -> do NOT load speaker contact policy
            - "apply" -> load speaker contact policy (global + override), merge all fields into group context
            - "restricted" -> load speaker contact policy but only use trust_level and forbidden_actions
         g. Build ResolvedPolicy using buildGroupPayload.
         h. Cache and return.
       - `resolveInboundPolicy(params: { isGroup: boolean; chatId: string; senderId: string; basePath: string }): ResolvedPolicy | null` — dispatcher:
         - If isGroup: return resolveGroupPolicy.
         - Else: return resolveContactPolicy using senderId as chatId (DM sender = contact).
         - Wrap in try/catch: on error log `[waha] rules resolution failed` and return null (non-fatal).
       - `resolveOutboundPolicy(params: { chatId: string; basePath: string }): ResolvedPolicy | null` — for outbound enforcement:
         - Determine if chatId is group (@g.us suffix) or contact.
         - If group: resolveGroupPolicy with senderId = "" (no speaker for outbound).
         - If contact: resolveContactPolicy.
         - Wrap in try/catch, return null on error.

    2. Write `tests/rules-resolver.test.ts`:
       - Use temp directories with seed YAML files for each test.
       - Test DM resolution: global default only, with override, with malformed override.
       - Test group resolution: all 4 allowlist modes, all 3 unknown_participant_policy values, all 3 contact_rule_mode values.
       - Test cache behavior: verify second call is fast (or mock policyCache).
       - Test resolveInboundPolicy dispatching (isGroup true/false).
       - At least 12 test cases total.

    IMPORTANT: Use fs.existsSync and fs.statSync for file existence and mtime checks. These are tiny files — no async overhead needed. Follow the error handling pattern from RESEARCH.md: never throw from resolvers, degrade gracefully.
  </action>
  <verify>
    <automated>npm test -- --reporter=verbose tests/rules-resolver.test.ts</automated>
  </verify>
  <done>DM and group resolution produce correct merged policies for all allowlist modes, contact_rule_modes, and unknown_participant_policies. Cache prevents redundant file reads.</done>

       - `buildDmPayload(params: { contactRule: ContactRule; targetId: string; actorId?: string; ownerId?: string; }): ResolvedPolicy`:
         - Returns `{ chat_type: "dm", target_id: targetId, can_initiate: contactRule.can_initiate, can_reply: contactRule.can_reply, privacy_level: contactRule.privacy_level, tone: contactRule.tone, language: contactRule.language, forbidden_actions: contactRule.forbidden_actions ??        - `buildGroupPayload(params: { groupRule: GroupRule; targetId: string; speakerId: string; speakerAllowed: boolean; actorId?: string; ownerId?: string; speakerContactRule?: Partial<ContactRule>; }): ResolvedPolicy`:
