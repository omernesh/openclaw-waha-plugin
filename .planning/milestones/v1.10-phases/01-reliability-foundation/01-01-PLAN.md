**Step 3: Create src/http-client.ts**

Extract `callWahaApi` from send.ts (lines 37-70) into this new module. The function signature EXPANDS to include reliability metadata:

```typescript
interface CallWahaApiParams {
  baseUrl: string;
  apiKey: string;
  path: string;
  method?: "GET" | "POST" | "PUT" | "DELETE";
  body?: Record<string, unknown>;
  query?: Record<string, string>;
  // NEW reliability fields:
  context?: { action?: string; chatId?: string };
  skipRateLimit?: boolean;
  timeoutMs?: number;
}
```

The new fields are all optional, so ALL existing 103 call sites in send.ts continue to work with zero changes.

Inside callWahaApi, layer these IN ORDER:
1. **Rate limit** (unless `skipRateLimit: true`) -- await `bucket.acquire()`
2. **Shared backoff check** -- if a 429 backoff is active, wait for it to clear
3. **Fetch with timeout** -- `signal: AbortSignal.timeout(timeoutMs ?? 30_000)`
4. **Timeout error handling** -- catch TimeoutError; for mutations (POST/PUT/DELETE) add "may have succeeded" warning; for reads (GET) the caller can retry
5. **429 handling** -- call `retryWith429Backoff()` which reads Retry-After header, applies exponential backoff with jitter (1s/2s/4s base, +/-25% jitter, cap 30s), sets shared `backoffUntil` timestamp, max 3 retries
6. **Error logging** -- on non-ok response, log `[WAHA] ${context.action ?? method} ${context.chatId ?? ""} failed: ${status} ${errorText}` via console.warn, then throw
7. **Response parsing** -- same as original (JSON if content-type matches, else text)

**Token Bucket class** (~60 lines): Custom implementation in http-client.ts.
- Constructor: `capacity` (default 20), `refillRate` (default 15 tokens/sec)
- `acquire()`: refill based on elapsed time, if token available decrement and return, else queue the resolve callback and schedule drain
- Use a single drain interval approach (per research recommendation) rather than per-request setTimeout
- Export the class for testing and for Phase 4 per-session buckets

**Shared backoff state**: Module-level `backoffUntil = 0` timestamp. `waitForBackoffClear()` checks and waits. `retryWith429Backoff()` sets it.

**warnOnError helper**: Export `function warnOnError(context: string)` that returns a catch handler logging `[WAHA] ${context}: ${err.message}`. This replaces `.catch(() => {})` patterns (used in Plan 02).
