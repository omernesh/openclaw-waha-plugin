    - Test: at level "warn", logger.info() produces no output; logger.warn() does
    - Test: at level "error", logger.warn() produces no output; logger.error() does

    - Test: default level is "info" when no config or env var set

  <action>
    1. Create src/logger.ts with:
       - Type `LogLevel = "debug" | "info" | "warn" | "error"`
       - Level priority map: debug=0, info=1, warn=2, error=3
       - `createLogger(opts?: { level?: LogLevel, component?: string, [key: string]: unknown })` factory
       - Logger interface: `{ debug, info, warn, error, child(fields) }` — each method takes (msg: string, extra?: Record<string,unknown>)
       - Each log call writes one JSON line to process.stdout (info/debug) or process.stderr (warn/error) via `JSON.stringify({ level, ts: new Date().toISOString(), component, msg, ...extra })` + newline
       - Level check: only emit if call level >= configured level
       - `child(fields)` returns new logger inheriting parent fields + adding new ones
       - Default singleton: `export const logger = createLogger()` — reads level from `process.env.WAHA_LOG_LEVEL` or defaults to "info"
       - Export `setLogLevel(level: LogLevel)` to allow runtime reconfiguration (used when config loads)

       - Add to knownKeys set in validateWahaConfig: add 'logLevel' alongside 'adminToken'
       - Add comment: `// Phase 35 (OBS-01): Structured log level. Overrides WAHA_LOG_LEVEL env var.`

    3. Create src/logger.test.ts with vitest tests covering all behaviors above.
       - Capture stdout/stderr via spy on process.stdout.write / process.stderr.write
       - Parse captured output as JSON and assert fields
