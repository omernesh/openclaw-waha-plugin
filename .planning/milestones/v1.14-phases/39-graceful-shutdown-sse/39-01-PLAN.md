# Phase 39, Plan 01: Graceful Shutdown & SSE Hardening

## Objective
Make server shutdown clean — in-flight requests drain, SSE connections close, timers don't hold the process open, and SSE client count is capped.

## Context
- @src/monitor.ts — all changes here

## Tasks

### Task 1: In-flight request tracking + drain on close (GS-01)
type: auto

Add an `inFlightRequests` counter that increments when a request starts and decrements when `res` finishes.
Modify `stop()` to wait for the counter to reach 0 with a 10s hard timeout before resolving.

**Done criteria:** `stop()` returns a Promise that resolves after all in-flight requests complete or 10s elapses.

### Task 2: SSE keep-alive .unref() + abort cleanup (GS-02)
type: auto

Call `.unref()` on the SSE keep-alive `setInterval` so it doesn't hold the process open.
In the abort signal handler, clear all SSE intervals and close all SSE client connections.

**Done criteria:** SSE keep-alive interval is `.unref()`'d. Abort handler cleans up all SSE state.

### Task 3: SSE client cap at 50 (OBS-03)
type: auto

Before adding a new SSE client, check if `sseClients.size >= 50`. If so, respond with HTTP 503 and return.

**Done criteria:** New SSE connections beyond 50 are rejected with 503.

### Task 4: Verify TypeScript compiles
type: auto

Run `npx tsc --noEmit` and confirm clean compilation.

**Done criteria:** Zero TypeScript errors.

## Verification
- TypeScript compiles clean
- All three requirements (GS-01, GS-02, OBS-03) addressed

## Success Criteria

- SSE cleanup in abort handler
