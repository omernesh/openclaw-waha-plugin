**SEC-01: Admin API Bearer token authentication.**

1. In `src/config-schema.ts`: Add `adminToken` field to the top-level waha config schema (NOT per-account — this is a global admin panel setting). Add it alongside existing fields in `WahaAccountSchemaBase` or at the waha root level if there's a separate root schema. Type: `z.string().optional()`. Also support env var `WAHA_ADMIN_TOKEN`.

2. In `src/monitor.ts`: Create a `requireAdminAuth` function near the other helpers (after line ~106):
```typescript
function requireAdminAuth(req: IncomingMessage, res: ServerResponse, cfg: CoreConfig): boolean {
  const token = cfg.adminToken ?? process.env.WAHA_ADMIN_TOKEN;
  if (!token) return true; // No token configured = no auth required (backward compat)
  const authHeader = req.headers.authorization;
  if (!authHeader?.startsWith('Bearer ')) {
    writeJsonResponse(res, 401, { error: 'Authorization required' });
    return false;
  }
  if (authHeader.slice(7) !== token) {
    writeJsonResponse(res, 401, { error: 'Invalid token' });
    return false;
  }
  return true;
}
```

3. Add `requireAdminAuth` check at the TOP of the admin route handling block (right after the `/health` check at line ~412, BEFORE any `/api/admin/*` route). Use a single guard:
```typescript
if (req.url?.startsWith('/api/admin/') && !requireAdminAuth(req, res, cfg)) {
  return;
}
```
This covers ALL admin routes with one check. Place it before the first `/api/admin/restart` handler.

4. The SSE endpoint (`/api/admin/events`) is also under `/api/admin/` so it gets auth automatically.

5. Do NOT add auth to `/health` (must remain public for monitoring) or the webhook path (has its own HMAC auth).

**SEC-04: Default webhook HMAC to random secret.**

  - Log the generated secret: `console.log(\`[waha] Auto-generated webhook HMAC secret for account ${account.accountId}: ${secret}\`)`.
  - Return the generated secret.
