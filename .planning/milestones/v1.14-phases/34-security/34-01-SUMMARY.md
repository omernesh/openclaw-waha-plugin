# Phase 34 Plan 01: Admin Auth + HMAC Default Summary

**Bearer token auth on all admin API routes with auto-generated HMAC webhook secrets when not configured**

## What Was Built

### SEC-01: Admin API Bearer Token Auth
- Added `adminToken` field to `WahaConfigSchema` (Zod) and `WahaChannelConfig` (TypeScript type)
- Created `requireAdminAuth()` middleware in monitor.ts that checks `Authorization: Bearer <token>`
- Token sourced from `config.channels.waha.adminToken` or `WAHA_ADMIN_TOKEN` env var
- Single guard at top of request handler covers all `/api/admin/*` routes including SSE
- `/healthz` and webhook path remain unauthenticated (public monitoring + own HMAC auth)
- When no token configured, auth is disabled (backward compatibility)

### SEC-04: Default Webhook HMAC Secret
- Modified `resolveWebhookHmacSecret()` to generate random 32-byte hex secret when no HMAC is configured
- Secret cached per-account in `autoGeneratedHmacSecrets` Map for process lifetime
- Generated secret logged at startup so operator can configure WAHA to send matching signature
- `webhookHmacKey: "disabled"` explicitly returns empty string, disabling HMAC verification
