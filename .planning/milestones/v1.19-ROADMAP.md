- Used getClient() pattern for new send.ts functions (matches current codebase convention)
- API key CRUD stays in ACTION_HANDLERS but removed from UTILITY_ACTIONS (admin-only)
- Labels placed in chats.md with WhatsApp Business caveat (not a separate file)
- LID/Calls/Policy actions folded into skills/media.md as Other Utilities section
- SKILL.md version field moved to metadata.version (skill-creator constraint)
- Evals test decision-making not live WAHA execution (agent selects action + params, no API access needed)
- Group admin actions refused by LLM from group chat — all admin tests routed via DM
- WAHA participants API requires [{id:'...'}] object array format, not string array
- deleteGroup returns 501 on NOWEB — use leave as functional equivalent
