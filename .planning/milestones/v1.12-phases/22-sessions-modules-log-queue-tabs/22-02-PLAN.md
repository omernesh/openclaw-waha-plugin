Replace the placeholder LogTab.tsx with a full implementation:

**Data fetching:** useEffect on refreshKey AND activeLevel AND searchQuery (debounced). Fetch via `api.getLogs({ lines: 300, level: activeLevel, search: searchQuery })`. Use AbortController for cleanup. The server returns `{ lines: string[], source: string, total: number }`.

**Level filter chips:** Row of 4 toggle buttons at the top: ALL, INFO, WARN, ERROR.
- Map to server values: ALL='all', INFO='info', WARN='warn', ERROR='error'.
- Active chip gets a filled/primary variant, others get outline variant.
- On click: set activeLevel state, which triggers re-fetch.
- DO NOT include a DEBUG chip — the server has no debug level.
- Use Button component with variant="default" for active, variant="outline" for inactive. Size="sm".

**Search box:** Input with placeholder "Search logs..." and a clear button (X icon) that appears when text is non-empty.
- Debounce search: only re-fetch after 400ms of no typing. Use a ref-based debounce timer.
- Clear button resets search and re-fetches immediately.

**Log display area:**
- Render lines in a scrollable div with fixed height (h-[calc(100vh-280px)] or similar, using overflow-y-auto).
- Each line: `<div>` with monospace font (font-mono text-xs), whitespace-pre-wrap, break-all.
- Color coding per line using regex pattern matching:
  - Lines matching `/error|fail|crash|exception/i` get class `text-destructive`
  - Lines matching `/warn|drop |skip|reject|denied/i` get class `text-yellow-500 dark:text-yellow-400`
  - All other lines get class `text-muted-foreground`

**Auto-scroll behavior:**
- Use a ref for the scroll container.
- On initial load (first fetch) and when user has NOT manually scrolled up: auto-scroll to bottom.
- Track `userScrolledUp` with a ref. Set to true when user scrolls and `scrollTop + clientHeight < scrollHeight - 50` (50px threshold).
- Reset `userScrolledUp` to false when user scrolls back to bottom (within 50px).
- After data loads, if `!userScrolledUp`, scroll container to bottom.
- Add a "Scroll to bottom" button (fixed at bottom-right of the log area) that appears when userScrolledUp is true. On click: scroll to bottom and set userScrolledUp to false.

**Stats line:** Below the log area, show: "Showing {lines.length} of {total} lines (source: {source})" in muted text.

**Loading state:** Show "Loading logs..." while fetching.

**Empty state:** If lines array is empty after fetch, show "No log entries found" with the current filter settings description.
